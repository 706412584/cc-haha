# v0.5.12

本次更新聚焦桌面端体验和 Solo 工作流，让 AI 多代理协作过程更可见、更稳定，也补齐了插件市场与逆向工程插件的一些可用性细节。

## 主要更新

### Solo 议会面板

- 新增 Planner / Reviewer / Critic 三角色状态面板。
- 支持待命状态，避免会议卡片在任务结束或历史恢复后消失。
- 新增流程链：Planner → Reviewer → Critic → Final plan。
- 新增最终执行计划卡片，方便直接查看综合后的落地方案。
- `CHANGES_NEEDED` 会展示阻塞意见和可执行动作，不再只显示一个状态标签。
- 长输出默认折叠阈值提高，减少关键信息过早隐藏。

### 历史会话恢复

- 从历史 `background_task` 消息恢复后台任务状态。
- 重新打开会话时，Solo 议会和任务 UI 更稳定，不会因为内存态丢失而空白。

### 工作区文件编辑反馈

- CLI agent 修改文件后，桌面工作区能及时刷新文件树、变更列表和诊断信息。
- 如果用户正在编辑同一个文件，会显示 agent 来源冲突提示，避免静默覆盖未保存内容。

### 插件与 MCP 体验

- 插件列表和详情页增加语言服务相关展示。
- 插件前置依赖提示更清晰。
- 逆向工程插件补充 APK hardening 技能，并更新多份技能说明。

## 验证

- 桌面类型检查通过。
- Solo Council、chatStore、插件相关测试通过。
- PR Quality / desktop / server / native / coverage 检查通过。

## 升级说明

- 无破坏性迁移。
- 已有会话、工作区和插件配置可继续使用。
- Solo 议会的新结构化展示只在新输出包含稳定标记时显示；旧输出仍会按原有文本状态兜底解析。
