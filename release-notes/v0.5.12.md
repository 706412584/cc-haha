# v0.5.12

> 本版本把 editor/LSP 底座正式推到可用形态：工作区编辑器能感知 agent 文件修改，插件页新增常用语言服务器一键安装/检测，终端复制体验补齐；同时合入 Solo A/B/C 计划闸门、MCP marketplace 安装状态修复、桌面 UX quick wins 和 reverse-engineering workflow 加固。

## 新功能

### Language Servers 一键安装与检测（#35）

- Settings → Plugins 新增 **Language Servers / Common language servers** 面板，覆盖常见前后端语言：Python/Pyright、JavaScript/TypeScript、Java、C、C++、Go、PHP、C#、Rust、Lua。
- 每个语言服务器显示本机检测状态、安装按钮、重检按钮与 guide 链接。安装命令只会打开到可见终端，用户按 Enter 后才真正运行。
- Python/Pyright、TypeScript、Intelephense 等 npm 安装型 LSP 增加 npm global bin 兜底：即使 Windows `%APPDATA%\npm` 没进 PATH，`npm install -g pyright` 成功后也能被检测到。
- 非 npm 工具（如 `clangd`、`gopls`、`rust-analyzer`、`jdtls`、`lua-language-server`）仍保持 PATH 探测，不被 npm 兜底误判。

### Agent 文件编辑桥（Phase 4，#29）

- CLI agent 的 Edit / Write / NotebookEdit / MultiEdit 成功完成后，桌面工作区面板会实时刷新文件树、changed 列表与 LSP 错误数。
- 如果正在编辑同名 buffer，会标记 `source: 'agent'` 冲突 banner，沿用 Reload / Keep mine / Open conflict view，不会静默覆盖用户未保存内容。
- 重复 tool result、失败编辑、disconnect / clearMessages / new-session cleanup 都已覆盖。

### Solo A/B/C 计划闸门（#34）

- Solo 模式在实现前加入 Planner → Reviewer → Critic 的 A/B/C plan gate，再进入 implement → test → review → land。
- 新增 plan-critic specialist，并把 composer/header 中的文案统一改成更短的 **Solo**（#31）。

### 终端复制/粘贴体验（#35）

- 集成终端支持 Ctrl+Shift+C / Ctrl+Insert 复制，Ctrl+Shift+V / Shift+Insert 粘贴。
- 有选区时裸 Ctrl+C 复制并清空选区；无选区时仍放行 SIGINT。
- 右键行为贴近 Windows terminal：有选区复制，无选区粘贴。

### Reverse-engineering workflow 加固（#32、#33）

- reverse-engineering 插件只保留已端到端 smoke verified 的 MCP server，减少安装后不可用入口。
- 增加 APK hardening skill 与 shell-driven workflow refactor，强化 RE 任务入口和先决条件提示。

## 改进

- **桌面 UX quick wins（#37）**：新增共享 Skeleton、Tooltip，改进 TabBar / Browser / MessageList / StreamingIndicator 的可访问性和 loading 表现。
- **Sidebar 拆分（#37）**：把 `Sidebar.tsx` 中的大量纯函数和展示组件拆到 `sidebarUtils.ts` / `sidebarComponents.tsx`，降低主组件复杂度。
- **Plugin prerequisites 测试补强（#36）**：补 `PluginPrerequisitesModal` 单测与 vitest setup，覆盖缺失依赖展示、重检、安装命令等行为。

## 修复

- **MCP marketplace 已安装状态（#38）**：已安装服务现在正确显示 installed 状态，避免重复安装入口误导。
- **MCP marketplace 必填 env（#38）**：安装前尊重 marketplace 声明的 required env，缺失时引导配置而不是直接失败。
- **Provider Fetch Models CORS（#28）**：模型拉取改走 server route，避免混合内容 / CORS 阻断。
- **LSP npm PATH 误判（#35）**：修复 `npm install -g pyright` 已成功但 PATH 未包含 npm global bin 时仍显示未检测到的问题。

## 升级注意

- LSP install 命令不会后台自动执行，只会注入到可见终端等待用户确认。
- 如果 npm global bin 不在 PATH，检测面板现在能识别 npm shim，但实际 shell 中直接运行命令仍可能需要用户把 `%APPDATA%\npm` / npm global bin 加进 PATH。
- Agent 文件编辑桥只刷新桌面工作区状态，不改变 CLI 工具实现，也不修改 vendored upstream 文件编辑工具。
